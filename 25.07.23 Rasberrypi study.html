라즈베리파이란 무엇인가

 

라즈베리파이 4B는 강력한 성능과 다양한 기능을 제공하는 소형 컴퓨터로, 브로드컴 BCM2711 쿼드코어 Cortex-A72(ARM v8) 64-bit SoC @ 1.5GHz를 탑재하고 있습니다. 메모리는 모델에 따라 1GB, 2GB, 4GB LPDDR4-3200 SDRAM으로 구성되며, 2.4/5GHz IEEE 802.11ac Wi-Fi, Bluetooth 5.0 및 BLE를 지원합니다. 또한, 기가비트 이더넷, 2개의 USB 3.0 포트, 2개의 USB 2.0 포트를 포함하고 있어 연결성이 우수합니다. 표준 40핀 GPIO 헤더를 통해 기존 모델과의 호환성을 유지하며, 2개의 micro-HDMI 포트를 통해 최대 4K 해상도를 지원합니다.

IPI DSI 디스플레이 포트, MIPI CSI 카메라 포트, 스테레오 오디오 및 컴포지트 비디오 포트를 갖추고 있어 멀티미디어 처리에 적합합니다. H.265(4Kp60 디코드), H.264(1080p60 디코드 및 1080p30 인코드)와 OpenGL ES 3.0 그래픽 기능을 지원하며, 저장장치 및 운영체제 로딩용으로 Micro-SD 카드 슬롯을 제공합니다. 전원은 5V DC USB-C 커넥터를 통해 공급되며, PoE 기능도 지원하지만 별도의 PoE HAT이 필요합니다. 작동 온도는 0~50°C로, 다양한 IoT 및 임베디드 응용 프로그램에 적합한 설계를 갖추고 있습니다.

 

라즈베리파이 연결 part

 

[라즈베리파이와 터치스크린 전원 연결]

라즈베리파이와 LCD 터치스크린의 전원 연결을 위해, 터치스크린 기판 뒷면에 위치한 5개의 핀 중 5V 핀과 GND 핀을 라즈베리파이의 GPIO 핀에 연결하였다. 구체적으로, 터치스크린의 5V 핀은 라즈베리파이 GPIO 핀의 2번 핀(5V)과, GND 핀은 GPIO 핀의 6번 핀(GND)에 각각 연결하여 안정적인 전원 공급을 구현하였다. LCD 터치스크린의 LCD라고 적힌 단자가 세워져 있는 부분에 케이블을 연결하였다.

 

[라즈베리파이와 아두이노 UNO 연결]

GPIO 핀을 이용한 UART 및 I2C 통신 설계를 사용하였다. 라즈베리파이와 Arduino UNO 간의 직렬 통신을 위해 UART와 I2C 통신 방식을 활용하였다. UART 통신에서는 라즈베리파이의 GPIO 핀 TXD(핀 8)를 Arduino UNO의 RX(핀 0)에, RXD(핀 10)를 TX(핀 1)에 연결하였다. 또한, 두 장치 간의 공통 접지를 위해 라즈베리파이의 GND(핀 6)를 Arduino UNO의 GND에 연결하였다. Arduino UNO는 USB 케이블을 통해 전원을 공급받으므로 추가적인 전원 연결은 필요하지 않다. 라즈베리파이의 GPIO 핀은 3.3V 논리 신호를 사용하며, Arduino UNO는 5V 논리 신호를 사용하는 차이를 극복하기 위해 레벨 시프터(Level Shifter)를 사용하여 안정적이고 안전한 데이터 송수신을 구현하였다.

I2C 통신에서는 라즈베리파이의 SDA 핀(핀 3)을 Arduino UNO의 A4 핀에, SCL 핀(핀 5)을 A5 핀에 연결하였으며, GND 핀(핀 6)을 Arduino UNO의 GND에 연결하여 공통 접지를 설정하였다. 이와 마찬가지로, 서로 다른 전압 레벨(라즈베리파이 3.3V, Arduino UNO 5V)로 인한 통신 오류와 손상을 방지하기 위해 레벨 시프터(Level Shifter)를 활용하였다. 이러한 설계를 통해 두 장치 간의 UART 및 I2C 데이터 통신이 신뢰성과 안정성을 확보한 상태로 원활히 이루어지도록 구현하였다.

[서보모터 연결]

아두이노 우노(Arduino Uno)를 활용하여 MG945 서보 모터 3개의 동시 제어 방법을 설계하였다. MG945는 우수한 토크 성능을 제공하며, PWM(Pulse Width Modulation) 신호를 통해 정밀한 각도 제어와 회전 동작이 가능하다. 이를 위해 아두이노 우노, MG945 서보 모터 3개, 외부 전원 공급 장치(5V DC), 점퍼 케이블 등으로 하드웨어를 구성하였다.

MG945 서보 모터의 신호 핀은 각각 아두이노 우노의 디지털 핀(6번, 7번, 8번)에 연결하였으며, 전원 핀은 외부 5V DC 전원 공급 장치에, 접지 핀은 외부 전원의 GND와 아두이노 GND에 공통으로 연결하였다. 이러한 구성으로 안정적인 전기적 접지를 형성하고, 3개의 서보 모터를 동시 제어하며 고정밀 동작을 구현할 수 있었다.

4. 전압 허용 범위와 쇼트 주의사항

  • 전압 범위: 대부분 모터는 정격 전압 ±10~20% 내에서 안정 작동

  • 전압이 낮으면: 출력 토크 부족, 멈춤

  • 쇼트 방지:

    • 전원선과 GND가 접촉하지 않게 주의

    • 모터 드라이버에 역전류 방지 다이오드, 커패시터 사용 권장

    • 전원은 모터용과 라즈베리파이용 분리하고 GND만 공통으로 연결

🔀 2. Broadcom vs Board 핀 번호 체계

라즈베리파이에서는 2가지 방식으로 핀을 식별할 수 있다.

 

예시 코드:

import RPi.GPIO as GPIO
import time


GPIO.setmode(GPIO.BCM)       # BCM 핀 번호 체계 사용
GPIO.setup(18, GPIO.OUT)     # GPIO 18을 출력으로 설정


try:
    while True:
        GPIO.output(18, True)
        time.sleep(1)
        GPIO.output(18, False)
        time.sleep(1)
except KeyboardInterrupt:
    GPIO.cleanup()

 

서보모터 제어 (GPIO 17)

xxxxxxxxxx
온도센서 (DHT11/22) 데이터 읽기
조도센서 + LED 밝기 조절
서보모터 제어


import RPi.GPIO as GPIO
import time


GPIO.setmode(GPIO.BCM)
GPIO.setup(17, GPIO.OUT)


servo = GPIO.PWM(17, 50)  # 50Hz PWM
servo.start(0)


def set_angle(angle):
    duty = 2 + (angle / 18)
    GPIO.output(17, True)
    servo.ChangeDutyCycle(duty)
    time.sleep(0.5)
    GPIO.output(17, False)
    servo.ChangeDutyCycle(0)


try:
    set_angle(0)
    time.sleep(1)
    set_angle(90)
    time.sleep(1)
    set_angle(180)
finally:
    servo.stop()
    GPIO.cleanup()

 

Flask로 LED 제어 웹서버

xxxxxxxxxx
라즈베리파이 <-> Android 앱 통신 (WebSocket)
MQTT로 센서 데이터 전송
Flask로 간단한 웹서버 구축


from flask import Flask
import RPi.GPIO as GPIO


app = Flask(__name__)
GPIO.setmode(GPIO.BCM)
GPIO.setup(18, GPIO.OUT)


@app.route('/led/on')
def led_on():
    GPIO.output(18, True)
    return "LED ON"


@app.route('/led/off')
def led_off():
    GPIO.output(18, False)
    return "LED OFF"


if __name__ == '__main__':
    app.run(host='0.0.0.0', port=5000)
